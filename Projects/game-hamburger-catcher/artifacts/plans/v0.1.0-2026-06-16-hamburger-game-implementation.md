## Architecture Overview & Execution Logic

The game is a single-page HTML5 Canvas-based application.

```
+-------------------------------------------------------------+
|                        HTML Container                       |
|  +-------------------------------------------------------+  |
|  |                    HUD (Overlay)                      |  |
|  |   Score: 120   Coins: 45   Basket: Wood (Lv.1)        |  |
|  +-------------------------------------------------------+  |
|  |                    Canvas Element                     |  |
|  |                                                       |  |
|  |    [Falling items: 🍔  🟩🍔  🟩🥔  🐔  🐄]           |  |
|  |                                                       |  |
|  |    [Player Basket] 🧺                                 |  |
|  |    ----------------------[Ground]-------------------- |  |
|  |           🪨 (Rock)             💧 (Water Pool)       |  |
|  +-------------------------------------------------------+  |
|  |                    Shop (Overlay)                     |  |
|  +-------------------------------------------------------+  |
+-------------------------------------------------------------+
```

### Tech Stack
- **Structure**: `index.html` (DOM elements, HUD, Shop layout)
- **Styling**: `style.css` (Warm Retro Arcade styling, Glassmorphism, animations)
- **Logic**: `game.js` (Canvas physics, spawning, Web Audio SFX)

### Phase 1. Core Physics & Ground Hazards ⚙️ `Difficulty: 🟡 Medium`

#### Implementation Plan
1.1 🤖 **Character Physics** — Implement character horizontal acceleration, friction, jumping gravity, and boundaries.
1.2 🤖 **Ground Generation** — Draw the ground platform with static Rocks (collision stuns player) and Water Pools (collision damage, resets position).

#### Task List
- [ ] 1.1 🤖 Implement keyboard and responsive touch inputs
- [ ] 1.2 🤖 Implement player velocity, jumping, and collision boxes
- [ ] 1.3 🤖 Add rocks (rendering, stun timer logic)

### Phase 2. Falling Items Engine ⚙️ `Difficulty: 🟡 Medium`

#### Implementation Plan
2.1 🤖 **Spawner Logic** — Create a spawning manager that drops items at adjustable intervals.
2.2 🤖 **Item Behaviors** — Implement different trajectories, size properties, and scores for regular/green burgers, green potatoes, chickens, cows, and big hamburgers (with erratic movements).

#### Task List
- [ ] 2.1 🤖 Implement spawning intervals and item speed variance
- [ ] 2.2 🤖 Implement regular hamburgers (straight drop, positive score)
- [ ] 2.3 🤖 Implement green hamburgers/potatoes (straight drop, negative score & health)
- [ ] 2.4 🤖 Implement chickens/cows (straight/flapping drop, large basket damage)
- [ ] 2.5 🤖 Implement big hamburgers (bouncing/diagonal paths, breaks basic basket)
