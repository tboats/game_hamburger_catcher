# Template: Paper (Formal Analysis)

> For formal analysis, case studies, or technical evaluations.
> Target length: 200–1000 lines.

## Standard Structure

```markdown
# [Paper Title]

> **Version:** X.Y.Z | **Updated:** YYYY-MM-DD
> **Source document:** [link to source]
> **Classification:** [Analysis / Case Study / Evaluation]

---

## Abstract

[3-5 sentences summarizing the problem, methodology, and key conclusions.]

---

## 1. Context & Motivation

[Why is this analysis needed? What problem is occurring?]

## 2. Methodology

[Approach to the analysis. Data sources, tools, process.]

## 3. Analysis

### 3.1 [Aspect 1]

[Detailed analysis with evidence (file paths, code, metrics)]

### 3.2 [Aspect 2]

[...]

## 4. Results & Evaluation

| Criteria | Result | Notes |
|:--|:--|:--|
| ... | ... | ... |

## 5. Conclusion & Recommendations

[Summary. Proposed next actions.]

## References

- [File 1](path/to/file)
- [File 2](path/to/file)
```

## Additional Rules

1. **Abstract required:** Paper MUST have an Abstract section at the top.
2. **Evidence:** Every claim in the Analysis section MUST have a specific reference (file path, line number, commit).
3. **References section:** MUST list all source files that were read.
4. **Concise:** Do not exceed 1000 lines. If more detail is needed, switch to Ebook format.
